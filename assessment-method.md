# Assessment Method

This field kit uses a simple consulting pattern:

1. Collect identity inventory.
2. Normalize the data.
3. Score enablement speed and access risk.
4. Identify gaps by control area.
5. Produce a remediation plan.
6. Create evidence records for what was reviewed.

## Assessment Domains

| Domain | What the kit checks |
|---|---|
| Identity lifecycle | Joiner, mover, leaver coverage and stale account indicators |
| Authentication | MFA readiness and policy coverage |
| Federation / SSO | App coverage and manual access patterns |
| Privileged access | Role assignment review, emergency access, owner clarity |
| Service accounts | Ownership, last sign-in, MFA exemption, expiry handling |
| Access reviews | Cadence, reviewer, evidence trail |
| MTTE | Estimated access enablement friction for critical apps |

## Scoring Model

Risk is scored from 0 to 100.

- 0–24: Low
- 25–49: Moderate
- 50–74: High
- 75–100: Critical

MTTE is scored separately because slow access is both a business problem and a security problem. If the process is slow, teams create workarounds.
